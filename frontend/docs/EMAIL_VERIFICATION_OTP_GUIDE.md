### Step 2: Configure Email Service

#### Option A: Gmail (Recommended for Testing)

1. **Enable 2-Factor Authentication** on your Gmail account
2. **Generate App Password:**
   - Go to: https://myaccount.google.com/apppasswords
   - Select "Mail" and "Other (Custom name)"

3. **Update `.env` file:**
```env

EMAIL_PORT=587
EMAIL_USER=your-email@gmail.com
EMAIL_PASSWORD=your-16-char-app-password
FRONTEND_URL=http://localhost:5173
```

**SendGrid:**
```env
EMAIL_HOST=smtp.sendgrid.net
EMAIL_PORT=587
```


## üîÑ Complete User Flows

### Flow 1: User Registration with Email Verification

```
1. User fills registration form
   ‚Üì
2. POST /api/auth/register
   - Creates user account
   - Generates verification token
   - Sends verification email
   ‚Üì
3. User receives email with verification link
   ‚Üì
4. User clicks verification link
   ‚Üì
5. GET /api/auth/verify-email/:token
   - Verifies email
   - Sends welcome email
   - Returns auth token
   ‚Üì
6. User is automatically logged in
   ‚úÖ Account fully activated
```

### Flow 2: Login from New Device with OTP

```
1. User enters email and password
   ‚Üì
2. POST /api/auth/login
   - Validates credentials
   - Detects device fingerprint
   - Checks if device is trusted
   ‚Üì
3. If NEW DEVICE:
   - Generates 6-digit OTP
   - Sends OTP email
   - Returns requiresOTP: true
   ‚Üì
4. User receives OTP email
   ‚Üì
5. User enters OTP
   ‚Üì
6. POST /api/auth/verify-otp
   - Validates OTP
   - Adds device to trusted list
   - Returns auth token
   ‚Üì
7. User is logged in
   ‚úÖ Device is now trusted
```

### Flow 3: Login from Trusted Device

```
1. User enters email and password
   ‚Üì
2. POST /api/auth/login
   - Validates credentials
   - Detects device fingerprint
   - Recognizes trusted device
   ‚Üì
3. User is immediately logged in
   ‚úÖ No OTP required
```

```http
  "password": "password123"
```

## üîê Security Implementation

### 1. Token Hashing
```javascript
// Email verification token
const token = crypto.randomBytes(32).toString('hex');
const hashedToken = crypto.createHash('sha256').update(token).digest('hex');
```

### 2. OTP Generation & Hashing
```javascript
// Generate 6-digit OTP
const otp = Math.floor(100000 + Math.random() * 900000).toString();
const hashedOTP = crypto.createHash('sha256').update(otp).digest('hex');
```

### 3. Device Fingerprinting
```javascript
// Generate unique device ID
const deviceId = crypto.createHash('sha256')
  .update(`${userAgent}-${ip}`)
  .digest('hex');
```

### 4. Expiration Times
- **Email Verification Token:** 24 hours
- **Login OTP:** 10 minutes
- **JWT Token:** 7 days (configurable)

## üêõ Troubleshooting

### Issue: Emails Not Sending

**Check:**
1. ‚úÖ Email credentials in `.env` are correct
2. ‚úÖ App password is generated (for Gmail)
3. ‚úÖ 2FA is enabled (for Gmail)
4. ‚úÖ Firewall allows SMTP port 587
5. ‚úÖ Check server logs for errors

**Test Email Connection:**
```javascript
// Add to server.js for testing
const nodemailer = require('nodemailer');
const transporter = nodemailer.createTransporter({
  host: process.env.EMAIL_HOST,
  port: process.env.EMAIL_PORT,
  auth: {
    user: process.env.EMAIL_USER,
    pass: process.env.EMAIL_PASSWORD
  }
});

transporter.verify((error, success) => {
  if (error) {
    console.log('‚ùå Email config error:', error);
  } else {
    console.log('‚úÖ Email server ready');
  }
});
```

### Issue: OTP Not Working

**Check:**
1. ‚úÖ OTP hasn't expired (10 minutes)
2. ‚úÖ Correct userId and deviceId sent
3. ‚úÖ OTP is 6 digits
4. ‚úÖ No typos in OTP

### Issue: Device Always Detected as New

**Check:**
1. ‚úÖ User-Agent header is being sent
2. ‚úÖ IP address is consistent
3. ‚úÖ Browser not in incognito mode
4. ‚úÖ VPN not changing IP
